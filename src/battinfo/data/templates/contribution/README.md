# BattINFO Dataset Contribution

Three steps to publish your battery measurement data to the BattINFO registry.

---

## Step 1 — Fill in the metadata

Open `battinfo.yaml` and fill in:

| Field | What to put |
|---|---|
| `cell_name` | A short label for this specific cell (serial number, lab code) |
| `cell_spec_iri` | The BattINFO IRI for the cell model — find it at [battery-genome.org/registry/cell-spec](https://www.battery-genome.org/registry/cell-spec) |
| `lab` | Your institution or lab name |
| `experimenter` | Optional — who ran the experiments |
| `license` | Keep `CC-BY-4.0` unless you have a reason to change it |

---

## Step 2 — Add your data files

Put your measurement CSV files in the `data/` folder.

**Name each file as:**
```
YYYY-MM-DD__testtype__temperature.csv
```

**Examples:**
```
data/
  2026-03-05__ici__25degC.csv          ← ICI test at 25 °C on 2026-03-05
  2026-02-20__rate__25degC.csv         ← Rate capability at 25 °C
  2026-03-06__capacity__25degC.csv     ← Capacity check at 25 °C
```

**Recognised test types:**

| Filename token | Test type |
|---|---|
| `ici` | Intermittent Current Interrupt |
| `rate` | Rate capability |
| `capacity` | Capacity check |

**Temperature format:** `25degC`, `40degC`, `-10degC`, etc.

Optionally, add cell photos (JPEG or PNG) to the `photos/` folder.

---

## Step 3 — Process and publish

```bash
# Process (validate + build the submission package)
battinfo dataset process .

# Upload to Zenodo (creates a draft deposit — you review before publishing)
battinfo dataset publish . --zenodo --token YOUR_ZENODO_TOKEN

# Or upload to Battery Genome directly
battinfo dataset publish . --registry --token YOUR_BATTINFO_TOKEN
```

Get a Zenodo token at https://zenodo.org/account/settings/applications/tokens/new/
(Tick: `deposit:write`)

---

## File layout

```
my-contribution/
  battinfo.yaml     ← metadata (edit this)
  README.md         ← this file
  data/             ← put CSV files here
  photos/           ← optional cell photos
  dist/             ← generated by `battinfo dataset process` — do not edit
```

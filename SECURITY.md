# Security Policy

## Supported versions

BattINFO is in beta. Security fixes are applied to the latest
`main` branch and the most recent release.

| Version | Supported |
|---------|-----------|
| 0.7.x | :white_check_mark: |
| < 0.7.0 | :x: |

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, use one of the following private channels:

- Open a [GitHub security advisory](https://github.com/BIG-MAP/BattINFO/security/advisories/new) (preferred), or
- Email **j.simon.clark@gmail.com** with the details.

Please include:

- A description of the vulnerability and its potential impact.
- Steps to reproduce, or a proof of concept.
- Affected version(s) or commit.

## What to expect

- We aim to acknowledge reports within **5 business days**.
- We will keep you informed of remediation progress and coordinate disclosure
  timing with you.
- We are happy to credit reporters in the release notes unless you prefer to
  remain anonymous.

Thank you for helping keep BattINFO and its users safe.
